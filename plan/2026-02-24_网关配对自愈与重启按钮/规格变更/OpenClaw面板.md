# OpenClaw面板 - 变更

## 新增需求

### 需求：网关配对自愈操作

面板必须支持读取并批准 Gateway 的 pending 设备配对请求，帮助用户在 `pairing required` 场景下无需命令行即可恢复连接。

#### 场景：一键批准待处理配对
- **当** 用户在仪表盘点击“批准待处理配对”
- **那么** 系统必须读取 pending 请求并依次执行批准，返回批准数量与失败详情

#### 场景：无 pending 的空操作
- **当** 当前没有待批准配对请求
- **那么** 系统必须返回“无需处理”的成功结果，不报错

### 需求：仪表盘网关快速重启动作

面板必须在仪表盘提供“重启网关服务”入口，调用既有服务控制链路并反馈执行结果。

#### 场景：一键重启网关
- **当** 用户在仪表盘点击“重启网关服务”
- **那么** 系统必须触发网关重启并反馈成功或失败原因
