# OpenClaw面板 - 变更

## 新增需求

### 需求：重置 Token 自动批准待处理配对

面板在执行“一键重置 Gateway Token”后，必须自动读取并批准 pending 设备配对请求，减少用户二次操作。

#### 场景：重置后自动审批 pending
- **当** 用户点击“一键重置 Gateway Token”并且存在待处理配对请求
- **那么** 系统必须在写入新 Token 后自动批准 pending 请求，并返回批准结果

#### 场景：重置后无 pending
- **当** 用户点击“一键重置 Gateway Token”但没有待处理配对请求
- **那么** 系统必须返回“无需处理”的成功结果，不报错
