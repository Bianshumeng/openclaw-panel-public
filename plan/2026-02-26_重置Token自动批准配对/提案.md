# 重置 Token 自动批准配对

创建时间：2026-02-26 05:28:00

## 为什么做

用户在 `gateway token mismatch` 场景下完成 Token 重置后仍会遇到 `pairing required`，当前流程要求手工再点击一次“批准待处理配对”，操作链路割裂且容易漏操作。

## 改什么

- 将 `/api/gateway/token/sync` 升级为复合动作：重置 Gateway Token 后自动执行 pending 配对审批。
- 仪表盘“重置 Token”动作完成后直接展示自动审批结果，减少人工排障步骤。
- 补充单元测试，覆盖复合动作的成功、无 pending、审批失败场景。

## 影响范围

- 涉及模块：`src/server.js`、`src/gateway-token.js`、`public/js/pages/model-dashboard-page.js`、`test/unit/gateway-token.test.js`
- 涉及接口：`POST /api/gateway/token/sync`
- 风险等级：L1

## 不做什么（边界）

- 本次不移除“手动批准待处理配对”按钮，仍保留独立兜底入口。
- 本次不修改 OpenClaw 上游配对机制与安全策略。
- 本次不引入跨页面改动，仅调整仪表盘链路。
