# OpenClaw面板 - 变更

## 新增需求

### 需求：重置后延迟配对自愈

面板在重置 Gateway Token 后，必须在短窗口内持续轮询并自动批准新出现的 pending 配对请求，覆盖“连接动作晚于重置动作”的时序问题。

#### 场景：重置后延迟出现 pending 仍能自动批准
- **当** 用户完成 Gateway Token 重置后在 2 分钟内发起连接，并产生新的 pending 配对请求
- **那么** 系统必须通过轮询自动批准该请求，并在仪表盘反馈成功结果

#### 场景：轮询窗口未命中 pending
- **当** 重置后轮询窗口内没有新的 pending 配对请求
- **那么** 系统必须返回“监听结束可手动批准”的提示，不报错中断
